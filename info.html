Chain of Custody (CoC) System Documentation

Overview

The CoC System is a lightweight digital Chain of Custody tool designed to securely store and manage evidence during digital forensic investigations. Supervisors can create cases and assign investigators to collaborate within a controlled and secure environment. Every action taken in the system is automatically logged to ensure transparency, accountability, and the integrity of the investigation process.

Supervisor Role

Supervisors oversee and manage investigations, ensuring all procedures follow established standards. Their responsibilities include:

  • Creating new cases.
  • Assigning investigators to cases.
  • Monitoring the progress of ongoing investigations.
  • Assigning existing evidence to different cases.
  • Reviewing case reports.
  • Closing cases when investigations are complete.

Supervisors act as coordinators and overseers, with full permissions to manage cases and maintain security, transparency, and credibility throughout the investigation process.

Investigator Role

Investigators are responsible for handling and submitting evidence within the cases assigned to them. Their main tasks include:

  • Viewing cases assigned by supervisors.
  • Uploading evidence files with relevant metadata.
  • Adding comments or notes related to submitted evidence.
  • Following the procedures to ensure evidence integrity.

Investigators work under the guidance of supervisors, ensuring all evidence is managed securely and every action is properly logged.

Key Features

  • User Authentication: Secure login system with hashed passwords.
  • Role-Based Access Control: Supervisors and investigators have distinct permissions.
  • Case Management: Supervisors can create, assign, and manage cases; investigators can view assigned cases.
  • Evidence Management: Upload artefacts with metadata (creation, modification, access times). Supervisors can approve or reject evidence.
  • Comments: Investigators can leave comments for supervisors on evidence submissions.
  • Chain of Custody Logs: Track actions such as uploads, downloads, rehashes, and assignments, with reports available in PDF or HTML format.
  • Blob Storage: Evidence files are stored securely using BLOB storage.
  • Metadata Extraction: Automatic extraction of available file metadata.
  • Administrative Features: Add new investigators, assign investigators to cases, and review pending evidence submissions.

Full Feature and Tool Index

The CoC System provides the following core features and tools:

  • User Management: Registration, login, role-based access control (supervisors/investigators).
  • Case Management: Create, assign, monitor, and close cases.
  • Evidence Management: Upload/download files, track metadata, approve/reject submissions.
  • Comments: Leave notes on evidence for communication between investigators and supervisors.
  • Chain of Custody Logs: Automatically record all actions on evidence (upload, download, assignment, rehash).
  • Report Generation: Export CoC logs in PDF or HTML formats.
  • Blob Storage: Secure storage of uploaded evidence.
  • Metadata Extraction: Automatic reading of file properties (creation, modification, access times).
  • Administrative Tools: Add new users, manage roles, review pending evidence.

Libraries and Frameworks Used

  • PHP: Server-side scripting for logic and database interactions.
  • MySQL: Database management for storing users, cases, evidence, and logs.
  • Bootstrap: Front-end styling and responsive layout.
  • JavaScript: For front-end interactivity and dynamic page updates.
  • Other Libraries: Optional PDF generation library (e.g., TCPDF or FPDF), and PHP file handling utilities.
